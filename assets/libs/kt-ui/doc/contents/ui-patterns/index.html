사용자 인터페이스(UI)의 화면 유형을 다음과 같이 구분된다.

  • Basic Pattern - 기본적인 다건 및 단건 처리 유형
  • List to Detail - 조회 결과 다 건으로(List) 보여주고 선택 건에 대하여 상세 페이지로 넘어가는 유형
  • Master/Detail- 한 페이지에서 Master와 Detail의 Set 로 구성되어 있는 유형
  • Dashboard - 한 페이지에서 여러 widget으로 구성되어 있는 유형

Basic Pattern

Type 개요
[BP-SD1] Single Detail(Separate Edit Page) 한 건 조회 후 별도 수정 및 추가화면으로 페지가 전환되는 유형
[BP-SD2] Single Detail(Save Form) 한 건을 한 화면에서 CRUD 처리하는 유형
[BP-MPD] Multi Popup Detail(Popup Edit) 다 건을 조회를 한 후 추가적으로 popup 창을 띄워 1건에 대한 CRUD를 처리하는 유형
[BP-MG] Multi Detail(Save Grid) 다 건에 대해 그리드를 이용하여 별도의 페이지 전환 없이 CRUD 가능한 유형

Single Detail(Separate Edit Page)

개요

1건을 조회 한 후 별도 Edit Mode 로 들어가 편집하는 유형

적용 Case

1건 조회 업무가 주 업무인 경우
1건에 대한 조회를 한 후 이에 대한 수정을 하는 경우
